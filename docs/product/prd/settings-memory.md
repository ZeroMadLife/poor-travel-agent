# 设置与记忆 PRD

## 1. 页面目标

让用户用一套可扫描的设置结构理解 Sage 的个人偏好、模型能力、数据与记忆、运行诊断；同时把需要即时决定的 Memory Proposal 放回主对话，而不是让用户定期访问一个“记忆收件箱”。

## 2. 用户与任务

1. 修改主题、显示工具过程和动效偏好；
2. 查看 Provider、模型、Skills、MCP 的当前状态；
3. 审阅、批准、拒绝、删除或导出已保存 Memory；
4. 查看上下文、Usage、会话和运行诊断；
5. 用 `⌘K / Ctrl+K` 快速跳转设置或页面动作。

## 3. 信息架构：S1

### 个人与体验

- 外观；
- Purpose / Soul；
- 语言与通知。

### 模型与能力

- Provider 与模型；
- Skills；
- MCP。

### 数据与记忆

- Memory；
- 上下文；
- Knowledge 来源；
- 隐私与导出。

### 运行与开发

- 会话；
- 工作区与 Git；
- Usage；
- Run 与诊断。

现有 `/settings/:section?` 路由继续兼容，先映射到分组和子项，再决定是否合并旧 section。

## 4. Memory 体验

### 4.1 边界

- Memory 是长期偏好、明确约定或稳定事实，不是 Wiki 页面副本；
- 模型推断只能产生 proposal；
- 用户批准后才进入 durable memory；
- 每条 Memory 展示来源、创建时间、revision 和可撤销入口；
- 公开门面永远不读取私人 Memory。

### 4.2 待审阅 Proposal

待批准 proposal 不在设置首页堆积。主对话 Facts Rail 在 proposal 到达时显示：

- 一句话候选内容；
- 来源消息/证据；
- 影响范围；
- 批准、拒绝、稍后处理。

点击“查看全部”才进入设置 Memory 页面。

### 4.3 已保存 Memory

设置 Memory 页面提供搜索、来源筛选、时间排序、删除和导出。删除是明确的 destructive action，必须显示影响范围，不可用简单切换伪装。

## 5. S3 命令面板

### 触发

- macOS：`⌘K`；
- Windows/Linux：`Ctrl+K`；
- 导航中的搜索图标和空态按钮也可打开。

### 命令分组

| 命令 | 是否有副作用 |
| --- | --- |
| 跳转主对话 / Knowledge / 设置 | 否 |
| 搜索会话 / 页面 / 节点 | 否 |
| 继续当前 Goal | 可能启动 run，需二次确认 |
| 导入来源 | 会创建 job，进入导入确认 |
| 打开 Memory Proposal | 否 |
| 批准/拒绝 proposal | 是，必须进入审阅确认 |

命令面板搜索结果应显示页面图标、命令名称、快捷键和风险标签。

## 6. 页面状态

| 状态 | 处理 |
| --- | --- |
| 空设置 | 显示分组导航和最小配置入口，不展示大面积空卡片 |
| 加载 | 只在对应 section 显示 skeleton，其他设置可继续导航 |
| 服务未装配 | 显示只读/尚未开放原因，不把按钮做成可点击假功能 |
| 保存中 | 行级 loading，禁止重复提交 |
| 冲突 | 保留本地草稿，提示 revision 已更新并要求重新加载 |
| 删除/导出 | 显示确认范围和结果 receipt |

## 7. 契约依赖

- Purpose/Soul：revision、可见性、读写和导出；
- Memory：列表、来源、proposal、approve/reject、delete/export；
- Provider/Model/Skill/MCP：当前只读或已有设置 API；
- Usage：真实 provider usage；
- Command search：会话、Wiki、节点的统一只读查询；
- audit receipt：保存和删除的可追踪结果。

前端不能从 Memory proposal 候选内容推断最终保存状态。

## 8. 响应式与可访问性

- 桌面：左侧分组导航，右侧内容最大宽度约 `960px`；
- 移动：顶部“全部设置”选择器，一次只显示一个 section；
- 每个设置行键盘可到达，label 与 control 正确关联；
- 命令面板打开后捕获焦点，Escape 关闭并恢复触发焦点；
- 危险操作需要文字确认，不依赖颜色区分。

## 9. 验收标准

- 用户能在 10 秒内找到模型、Memory、Usage 和工作区设置；
- 待批准 Memory 在主对话可见，不需要用户主动刷新设置页；
- 已保存 Memory 具备来源、revision、删除和导出能力（契约开放后）；
- 未开放能力明确只读，不出现点击无反馈；
- `⌘K/Ctrl+K` 可打开、搜索、键盘选择、关闭并恢复焦点；
- 移动端无横向滚动和长行截断；
- 设置页不创建第二套聊天或 Knowledge 数据请求编排。

## 10. 成功指标

- 设置任务完成率和查找时间；
- Memory proposal 从出现到决定的时间；
- 冲突/失败后的安全恢复率；
- 命令面板使用率和命令成功率；
- 用户主动导出/删除长期信息的可发现性。
